# About

I'm a senior engineer working on Unreal Engine 5 game development, AI agent integrations, and full-stack web. I like reading large codebases more than starting from scratch, picking the right architectural pattern from prior art, and being honest about what's mine vs. what I'm building on.

Most of my recent work is on UE5 — cinematic trailer production, custom plugin development, AI-driven tooling, and engine-level C++. I evaluate commercial frameworks before adopting them, write down what I find, and ship custom extensions where the upstream falls short.

## What I do well

- **Read fast, judge well.** I can map a 400-file commercial UE5 plugin in an afternoon and tell you what's worth keeping, what's broken, and what to write yourself.
- **Architectural pattern recognition.** Interface-spine module graphs, anim-notify-driven action lifecycles, procedural-over-keyed animation — I recognize when a codebase has these and can extend them without breaking the invariants.
- **Engine-level C++.** Module structure, UBT toolchain quirks, editor subsystem patterns, the MSVC compiler-version pitfalls that block builds at 5pm on a Friday.
- **AI agent integration.** Building ACP / MCP / tool-call bridges that let language models drive real software safely — typed tool surfaces, scope-limited mutations, fail-fast error paths.
